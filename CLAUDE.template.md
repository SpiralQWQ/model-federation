| **Gemini (安全合规官)** | gemini-3.5-flash | **绝对不参与业务逻辑构思。**仅在最后一步对前三方合并的代码/方案进行跨平台兼容性测试与隐蔽安全漏洞扫描，输出最终的「合规审计签章」。 |

### 执行协议

1. DeepSeek 先发布初始方案
2. Qwen + GLM 同时独立审查（不互相污染）
3. DeepSeek 融合 Qwen + GLM → 形成三方共识方案
4. Gemini 对最终方案执行安全合规盲审 → 输出签章
5. **四方铁律**：DS → Qwen → GLM → Gemini **四缺一不可**。Gemini 若报错（如 400），必须自动重试或自行修正 Payload 后重试，**严禁以降级为由跳过 Gemini**。最多重试 3 次，3 次全败则在日志中记录 `Gemini exhausted retries` 并继续。
6. **模型名修正**：Gemini Flash 模型名为 `gemini-3.5-flash`（非 `gemini-3.5-flash`，后者不存在，已验证 API /models 列表）。
7. **若任一方提出 Blocker → 强制第二轮融合讨论**
