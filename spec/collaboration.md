# 协同协议 — 三方/四方协作规则

> 模型联邦 V7.0 多模型协作标准。适用于大规模重构、架构变更、跨文件依赖分析等需要多模型交叉验证的场景。

## 引擎分工

| 模型 | 角色 | 职责 |
|------|------|------|
| DeepSeek (主控) | 仲裁者 | 意图嗅探、分歧仲裁、最终决策、生成日志 |
| QwenMax | 外科医生 | 代码实作、精确修改、语法检查、预审查 |
| GLM-4.7Flash | 检查员 | 语义检查、质检评分、一致性验证 |
| Gemini 3.5 Flash | 审计员 | 架构审查、安全审计、边界条件验证 |

## 触发条件

- **显式触发**：用户提及「三方」「四方」「协同」「会诊」
- **隐式判定**：一次修改 >= 3 个文件；架构/路由/配置变更；百度优化目录下任何写入操作
- **禁止绕过**：上述场景未走协同视为违规

## 串行管线（默认流程）

```
DeepSeek (意图分析)
  → QwenMax (实作/改写)
  → GLM (语义质检 + 评分)
  → DeepSeek (结果审查 + 仲裁)
  → Gemini (架构审计 + 安全扫描)
  → DeepSeek (终审 + 签发日志)
```

- 每个节点等待前一个节点完成后再启动
- 任一节点返回 Blocker → 停止管线 → 进入融合讨论
- 融合讨论后仍无法解决 → 仲裁者 DeepSeek 行使最终否决权

## 物理日志铁律

**Log = Truth。** 每次协同必须生成物理日志文件到 `<YOUR_WORKSPACE>/模型协同日志/`。

- 日志文件名：`协同_YYYYMMDD_HHMM_<主题>.md`
- 内容必须包含：触发场景、参与模型、各引擎输出摘要、分歧点及裁决、最终决策
- 禁止无日志完成协同 — 无日志视为未执行

## 防作弊透传协议

代码修改场景使用 `--prompt-file` 指针传参：

```
<workflow command> --prompt-file <payload.txt>
```

- payload 文件写入 `<YOUR_WORKSPACE>/temp/payload_<timestamp>.txt`
- 避免命令行直接传递超长代码（>500 字符）
- payload 包含：上下文摘要、修改目标、约束条件

## 仲裁规则

1. 多数优先：至少两方同意可执行
2. Blocker 覆盖：任一模型提出 Blocker → 强制第二轮融合讨论
3. 最终否决：2v1 不能覆盖 Blocker；DeepSeek 仲裁者有最终决定权
4. 均衡器：Gemini 因故不可用时，Claude + QwenMax 一致视为通过

## 降级兜底

| 故障 | 处理方式 |
|------|---------|
| Gemini 不可用 | 记录故障日志，Claude + QwenMax 一致同意视为通过 |
| QwenMax 不可用 | 记录故障日志，Claude + Gemini 一致同意视为通过 |
| GLM 不可用 | 跳过语义质检，记录警告 |
| 连续 2 次全部失败 | FATAL_ERROR，挂起会话，请求人类介入 |
