# 日志模板 — 命名规则、存储位置与内容要求

> 所有协同日志、程序员日志必须按本规范存储。Log = Truth，无物理日志的协同视为未执行。

## 日志存储位置

| 日志类型 | 路径 |
|---------|------|
| 三方/四方协同日志 | `<YOUR_WORKSPACE>/模型协同日志/` |
| 程序员日志 | `<YOUR_WORKSPACE>/程序员日志/` |

## 命名规则

格式：`<序号>_<主题>_<YYYYMMDD>_<HHMM>.md`

- **序号**：两位数（01-99），同一主题按时间顺序递增
- **主题**：英文或中文简述，不超 6 字
- **YYYYMMDD**：8 位日期
- **HHMM**：4 位时间（24 小时制，必填）

示例：`01_日志采集_20260718_2300.md`

## 内容要求

### 协同日志（三方/四方）
```
# 协同日志 — <主题>

- 时间：YYYY-MM-DD HH:MM
- 参与模型：[DeepSeek / QwenMax / GLM / Gemini]
- 触发场景：<修改类型、涉及文件>

## 各引擎输出
- QwenMax：<摘要>
- GLM：<评分 + 检查结果>  
- Gemini：<审计结果>

## 分歧与裁决
- 分歧点：<如有>
- 裁决结果：<最终决策 + 理由>

## 变更摘要
- 改动的文件及内容简述
```

### 程序员日志
```
# 程序员日志 — <日期>

- 工作区：<工作区名>
- 主要任务：<核心任务描述>
- 模型使用：<模型名及调用次数>

## 工作记录
- HHMM - <具体工作描述>
- HHMM - <具体工作描述>

## 关键决策
- <决策项>：<决策内容 + 理由>

## 待办
- [ ] <未完成事项>
```

## 反偷懒审计中断协议

完成协作或日志记录后，必须**重新读取一次刚写入的物理日志文件**并对比预期内容：

1. Read 刚写入的日志文件
2. 验证：内容与执行过程一致、时间戳正确、参与模型已列出
3. 若发现内容不完整 → 补充缺失部分后再次验证
4. 审计不通过 → 标记日志为 `[UNVERIFIED]`，不得作为正式记录
