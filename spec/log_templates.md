# 协同日志模板 — 管线阶段

> **所属管线**：Phase 2（执行）
> **入口条件**：三方/四方协同完成（Phase 2 步骤全部通过）
> **用途**：协同日志命名、存储位置与内容要求。Log = Truth，无物理日志的协同视为未执行。

---

## 日志存储

所有协同日志统一存入 `<YOUR_WORKSPACE>/模型协同日志/`

## 命名规则

`协同_YYYYMMDD_HHMM_<主题>.md`

- **YYYYMMDD**：8 位日期
- **HHMM**：4 位时间（24 小时制，必填）
- **主题**：英文或中文简述，不超 6 字

示例：`协同_20260719_1430_三方审查.md`

## 日志内容模板

```
# 协同日志 — <主题>

- 时间：YYYY-MM-DD HH:MM
- 参与模型：[DeepSeek / QwenMax / GLM / Gemini]
- 触发场景：<修改类型、涉及文件>

## 各引擎输出
- QwenMax：<摘要>
- GLM：<评分 + 检查结果>
- Gemini：<审计结果>

## 分歧与裁决
- 分歧点：<如有>
- 裁决结果：<最终决策 + 理由>

## 变更摘要
- 改动的文件及内容简述

## Token 消耗
📊 [联邦计费] 模型1(输入→输出) + 模型2(输入→输出) = 总计 XXXX Token
```

## 反偷懒审计

完成协作日志后，必须重新读取刚写入的日志文件并对比预期：

1. Read 刚写入的日志文件
2. 验证：内容与执行过程一致、时间戳正确、参与模型已列出
3. 若发现内容不完整 → 补充缺失部分后再次验证
4. 审计不通过 → 标记日志为 `[UNVERIFIED]`，不得作为正式记录

---
## 质检门禁（Phase 3 — Qwen + GLM 各自独立审查，均≥9.5 方通过）

### 检查项细则

| 检查项 | 具体判定标准 | Qwen 评分 | GLM 评分 |
|--------|------------|:---------:|:--------:|
| 内容完整 | 包含：时间/参与模型/触发场景/各引擎输出/分歧裁决/变更摘要。缺任一 → 扣 3 分 | /10 | /10 |
| Token 计费已记录 | 日志尾部含联邦计费行。缺少 → 扣 2 分 | /10 | /10 |
| 审计已执行 | 日志未被标记 `[UNVERIFIED]`。未读验证直接提交 → 扣 4 分 | /10 | /10 |

### 通过条件

```
Qwen 评分 ≥ 9.5 AND GLM 评分 ≥ 9.5 → [质检] 裁决: 通过
Qwen 评分 < 9.5 OR GLM 评分 < 9.5 → [质检] 裁决: 返工（标注扣分项）
```
