# 视觉分析 — 双规交叉验证规则

> 图片分析场景的强制处理流程。使用 Qwen-VL-MAX + GLM-4.6V-Flash 双模型交叉验证，禁止仅用内置 Read 工具读取图片。

## 触发条件

- 用户提及「看图」「截图」「这张图」「分析图片」
- 附带了图片文件（PNG/JPG/WebP）
- 涉及 UI 截图、数据可视化图表、流程图、架构图

## 双规分析流程

1. **调用 vision_analyzer.py**（路径：`<YOUR_WORKSPACE>/tools/vision_analyzer.py`）
   ```
   python <path>/vision_analyzer.py "<图片路径>" "<分析提示词>" parallel
   ```
2. 工具内部并行调用 **Qwen-VL-MAX** + **GLM-4.6V-Flash**
3. 两个模型独立输出分析结果

## 结果融合

| 情况 | 处理 |
|------|------|
| 两模型结论一致 | 采纳共同结论 |
| 结论分歧 | 取交集共识 + 标注分歧点供用户确认 |
| 单一模型不可用 | 退化为单模型结果并记录警告 |

## 提示词编写指南

- 明确指定关注点（如「图中红框标注的部分是什么？」）
- 避免模糊描述（如「描述这张图」→ 改为「列出图中所有的 UI 元素及其功能」）
- 对于图表/数据类图片，要求提取具体数值而非仅描述趋势

## 禁止行为

- 禁止直接使用内置 Read 工具读取图片 — 必须通过 vision_analyzer.py
- 禁止只看单模型结果就得出结论
- 禁止跳过提示词直接让模型「分析图片」— 每次必须明确分析目标
